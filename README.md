# Spatial Waste Classification and Mapping using CNN Architectures

## Project Overview
This project is focused on spatial data mining applications, specifically utilizing simple Convolutional Neural Network (CNN) architectures to map and classify spatial areas. The primary objective is to perform a binary classification on spatial tiles to identify and differentiate between areas containing waste (Sampah) and areas without waste (Non-Sampah). The models assign a specific class label of 1 for identified waste and a class label of 0 for non-waste regions. The project evaluates, tests, and compares the training dynamics, system stability, and spatial mapping accuracy of two distinct CNN architectures: LeNet and AlexNet.

## Model Architectures and Training Dynamics

### 1. LeNet Architecture Performance
The LeNet model represents a relatively simpler CNN architecture, consisting primarily of two convolutional layers followed by pooling mechanisms and fully connected layers. 
* Throughout the 20 training epochs, the model demonstrated an overall decrease in training loss, indicating that the system was successfully learning to recognize basic data patterns and continuously reducing the amount of discarded, unrecognized data.
* However, the training process exhibited significant instability and fluctuations between underfitting and overfitting states at very specific epochs.
* For instance, at Epoch 19, the model showed clear signs of severe overfitting, recording a low training loss of 0.5085 but a highly inflated validation loss of 0.9866, meaning it completely failed to generalize its learning to unseen data.
* Conversely, Epoch 20 demonstrated underfitting on the training set with a high training loss of 0.5004, while simultaneously recording an unusually low validation loss of 0.1430.
* When deployed for actual spatial inference, the generated probability map revealed that the LeNet model heavily overgeneralized the environmental data. 
* Due to its simplistic and shallow architecture, LeNet misclassified many non-waste areas as waste, failing to distinguish complex spatial features accurately and producing a highly saturated classification map.

### 2. AlexNet Architecture Performance
AlexNet is designed as a much more complex and deeper architecture compared to LeNet, featuring a significantly greater number of convolutional and fully connected layers. 
* This increased structural depth provides AlexNet with a higher capacity to learn and extract highly complex, fine, and abstract feature representations from the spatial datasets.
* The training phase for AlexNet was characterized by highly volatile fluctuations in both training and validation loss across the entire 20 epochs.
* The model exhibited massive instability early on, with Epoch 1 showing a training loss of 0.6212 alongside a massive validation loss of 2.7490, indicating severe initial overfitting as the complex model struggled to initialize.
* Random underfitting and overfitting states were continuously observed and scattered throughout the training process (e.g., distinct underfitting at epochs 2, 3, 4, 10, 16, and 20).
* This consistent instability is directly attributed to AlexNet's complex structure, which makes it highly sensitive to complex data distributions, unbalanced data, and requires significantly more computational effort to optimize and stabilize during training.
* Despite the unstable training metrics, the final spatial probability map generated by AlexNet was vastly superior and more precise than LeNet's output.
* AlexNet successfully differentiated between waste and non-waste objects with high specificity, completely avoiding the broad overgeneralization seen in the LeNet results because it could properly handle the abstract features of the waste objects.

## Inference Pipeline and Methodology
The project includes a robust inference pipeline built with PyTorch and Rasterio to process spatial imagery, run model predictions, and generate GeoTIFF classification maps.
* The Python script iterates over split image tiles, opening the raster data and safely extracting the RGB channels (indexes 1, 2, and 3).
* Pixel values are mathematically normalized to a 0-1 floating-point range to correctly prepare the raw data for neural network ingestion.
* For the AlexNet model specifically, images are explicitly resized to exact 227x227 pixel dimensions using Python Imaging Library (PIL) and PyTorch transforms before being converted into multi-dimensional Tensors.
* PyTorch Tensors are strictly utilized over standard NumPy arrays because Tensors offer optimized multidimensional operations, deep integration with machine learning workflows, and native support for hardware execution on GPUs.
* The classification outputs from the model are passed directly through a Sigmoid activation function. 
* The Sigmoid function forms a mathematical S-curve that maps the non-linear network outputs to a strict probability range between 0 and 1, which is the required format for binary classification tasks.
* A strict programmatic threshold of 0.5 is applied to the Sigmoid probability output; values above the threshold are confidently classified as waste (1), and values below are designated as non-waste (0).
* The resulting binary classification is mapped to a zeroed 32x32 array, structurally formatted to match the spatial coordinates, and exported as a new GeoTIFF raster file containing the finalized spatial predictions.

## Core Concepts Addressed
* **Model State Management:** The project leverages PyTorch capabilities to save the model's `state_dictionary` (the architectural weights and trained parameters) into `.pth` files. This allows for rapid model loading (`Load weight` and `Load Model`) without the heavy computational need to retrain the network from scratch.
* **Segmentation Paradigms:** While this project focuses on foundational classification, it addresses broader computer vision concepts. It differentiates between Image/Semantic Segmentation (grouping pixels strictly by visual class, like 'waste' vs 'non-waste') and Panoptic Segmentation (which unifies background pixel classification with precise individual object instance detection).
* **Activation Functions:** The project strategically employs the Sigmoid function for its probabilistic binary outputs, contrasting it with Argmax, which is typically reserved for multinomial classification where the goal is to extract the single highest probability class index from a larger vector.
